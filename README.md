# spark-assets — хостинг картинок для лендингов (GHL)

Это **публичный** репозиторий ТОЛЬКО под картинки. Никакого кода/стратегии — только обезличенные ассеты.
Каждый файл здесь автоматически доступен по постоянному CDN-адресу (jsDelivr), который GHL подхватывает как обычную картинку.

## Формула адреса

```
https://cdn.jsdelivr.net/gh/Android64/spark-assets@main/<путь к файлу>
```

Примеры:
- `screens/spark-system.png` → `https://cdn.jsdelivr.net/gh/Android64/spark-assets@main/screens/spark-system.png`
- `photos/andrey-van.jpg`    → `https://cdn.jsdelivr.net/gh/Android64/spark-assets@main/photos/andrey-van.jpg`

Адрес = база + тот же путь, что и в репозитории. Структуру папок менять не нужно.

## Как добавлять картинки (всегда одинаково)

**Вариант 1 — мышкой, без терминала (проще всего):**
1. Открой репозиторий на github.com → нужную папку (`screens` / `photos` / `memes` / `assets`).
2. **Add file → Upload files** → перетащи картинки → **Commit changes**.
3. Готово. Картинка сразу живёт по адресу из формулы выше.

**Вариант 2 — через терминал:**
```bash
cd ~/Documents/Cowork/SPARK/spark-assets
# скопировал новые файлы в нужные папки, затем:
git add -A && git commit -m "add images" && git push
```

## Правила имён
- Имена файлов — **латиницей**, без пробелов и кириллицы (в URL это ломается). Дефисы вместо пробелов.
- Лучше **не перезаписывать** существующий файл тем же именем: jsDelivr кэширует версию ветки до ~12 часов, и старая картинка может ещё показываться. Хочешь заменить — либо новое имя (`photo-v2.jpg`), либо сбрось кэш, открыв один раз в браузере:
  `https://purge.jsdelivr.net/gh/Android64/spark-assets@main/<путь>`
- Новые файлы (новые имена) появляются на CDN **сразу**, без задержки.

## Папки
- `screens/` — скриншоты систем, CRM, отзывы-скрины
- `photos/`  — личные фото
- `memes/`   — мемы
- `assets/`  — иконки оплаты и прочая мелочь (`assets/pay/…`)
